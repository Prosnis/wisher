1. Firebase Authentication (Аутентификация)
Firebase Authentication позволяет управлять процессом аутентификации пользователей в приложении.

getAuth(): Инициализирует и возвращает объект аутентификации, который используется для вызова других методов аутентификации.

Регистрация:

createUserWithEmailAndPassword(auth, email, password): Регистрирует пользователя с помощью электронной почты и пароля.
Авторизация:

signInWithEmailAndPassword(auth, email, password): Авторизует пользователя с помощью email и пароля.
signInWithPopup(auth, provider): Авторизация через социальные сети (Google, Facebook и др.) через popup.
signInWithRedirect(auth, provider): Авторизация через социальные сети через редирект.
Работа с текущим пользователем:

currentUser: Свойство, возвращающее текущего авторизованного пользователя.
signOut(auth): Выход пользователя из аккаунта.
onAuthStateChanged(auth, callback): Подписка на изменение состояния аутентификации (логин/логаут).
Дополнительно:

sendPasswordResetEmail(auth, email): Отправляет пользователю письмо для сброса пароля.
updateProfile(user, profileData): Обновляет информацию о профиле (имя, фото).
2. Firebase Firestore (NoSQL база данных)
Firestore — это облачная база данных, которая хранит данные в виде документов и коллекций.

Инициализация:

getFirestore(): Инициализация Firestore, возвращает объект базы данных.
Чтение данных:

getDoc(docRef): Получает один документ по его ссылке.
getDocs(collectionRef): Получает все документы из коллекции.
onSnapshot(docRef, callback): Реализует реальное время прослушивания изменений в документе.
Запись данных:

setDoc(docRef, data): Создает или перезаписывает документ с данными.
addDoc(collectionRef, data): Добавляет новый документ в коллекцию с автогенерированным ID.
updateDoc(docRef, data): Обновляет существующий документ с новыми данными.
Удаление данных:

deleteDoc(docRef): Удаляет документ по его ссылке.
Создание ссылок на документы/коллекции:

doc(db, 'collection', 'docId'): Создает ссылку на документ.
collection(db, 'collectionName'): Создает ссылку на коллекцию.
3. Firebase Realtime Database (Реальное время)
Realtime Database — это другая база данных Firebase, которая хранит данные в виде JSON и синхронизирует их в реальном времени.

Инициализация:

getDatabase(): Инициализирует базу данных реального времени.
Чтение данных:

ref(db, path): Создает ссылку на данные по указанному пути.
get(ref): Получает данные по указанному пути.
onValue(ref, callback): Подписка на изменения данных в реальном времени.
Запись данных:

set(ref, data): Записывает данные по указанному пути.
update(ref, data): Обновляет данные по указанному пути.
push(ref, data): Добавляет данные с уникальным ключом.
Удаление данных:

remove(ref): Удаляет данные по указанному пути.
4. Firebase Storage (Хранение файлов)
Firebase Storage используется для хранения файлов, таких как изображения, видео и другие медиаданные.

Инициализация:

getStorage(): Инициализация Firebase Storage.
Работа с файлами:

ref(storage, path): Создает ссылку на файл.
uploadBytes(ref, file): Загружает файл в Storage.
getDownloadURL(ref): Получает URL для скачивания загруженного файла.
deleteObject(ref): Удаляет файл по ссылке.
5. Firebase Cloud Functions (Облачные функции)
Cloud Functions — это серверная логика, которая исполняется в облаке в ответ на события.

Создание облачных функций:
functions.https.onRequest(handler): Создает HTTP-функцию, которая вызывается при HTTP-запросах.
functions.firestore.document('path').onWrite(handler): Запускает функцию при изменении документа в Firestore.
6. Firebase Cloud Messaging (FCM)
FCM используется для отправки пуш-уведомлений на устройства.

Отправка сообщений:

messaging.getToken(): Получает токен для подписки устройства на уведомления.
messaging.onMessage(callback): Слушает входящие уведомления, когда приложение активно.
Отправка сообщений из сервера:

Для отправки сообщений требуется настроить серверную сторону (например, с использованием Firebase Admin SDK).
7. Firebase Analytics
Firebase Analytics отслеживает поведение пользователей в вашем приложении.

Инициализация:

getAnalytics(): Инициализация аналитики Firebase.
Отправка событий:

logEvent(analytics, 'event_name', eventParams): Логирование пользовательских событий (например, кликов, переходов).
8. Firebase Remote Config
Remote Config позволяет динамически изменять поведение и внешний вид приложения без необходимости публикации нового релиза.

Получение конфигураций:
getRemoteConfig(): Инициализация Remote Config.
fetchAndActivate(): Загружает и активирует новые конфигурации с сервера.
getValue(remoteConfig, key): Получает значение удаленной переменной по ключу.
9. Firebase Hosting
Firebase Hosting предоставляет инфраструктуру для развертывания веб-приложений.

Команды CLI:
firebase deploy: Разворачивает ваше приложение на Firebase Hosting.
firebase serve: Локально развертывает приложение для тестирования перед публикацией.
10. Firebase App Check
App Check помогает защитить ваше приложение от злоумышленников.

Инициализация:
initializeAppCheck(app, options): Инициализация App Check для защиты ваших ресурсов (базы данных, хранилища и др.).
11. Firebase Performance Monitoring
Этот сервис позволяет отслеживать производительность приложения.

Инициализация:

getPerformance(): Инициализирует мониторинг производительности.
Отслеживание производительности:

trace(name): Создает пользовательский трейс для мониторинга.
